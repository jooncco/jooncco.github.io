<br/><br/><br/><br/>

정수의 <span class="paragraph_bolster">이진수 표현</span>을 자료구조로 사용한다.

집합 $$ \{ A, B, C, D, E \} $$ 의 부분집합 $$ \{ C, E \} $$를 표현한다고 하면<br/>

$$ 00101_{(2)} $$ 이런식이다.

<br/><br/>

## 이점
> 빠른 연산속도

대부분의 연산이 <span class="paragraph_bolster">$$ O(1) $$</span>의 복잡도를 갖는다.<br/>
ex) 특정 원소의 존재여부 판단시 선형탐색할 필요 없이 AND 연산 결과가 0보다 큰지 검사<br/>

> 적은 메모리 사용량

vector\<bool\>의 <span class="paragraph_bolster">bool 타입</span>은 1바이트가 할당되지만,<br/>
true 혹은 false를 저장하기 위해 1 비트만 사용되므로 7비트 공간은 낭비를 하고 있는 셈이다.


> 배열을 정수로 대체하므로 index로 활용 가능

표현하려는 부분집합이 어떤 <span class="paragraph_bolster">탐색공간(상태)</span>을 나타내고, 그때의 값을 "cache" 변수에 저장하고 있다고 하자.
<span class="paragraph_little_chat">(탐색공간이  뭔지 와닿지 않는다면, 맨 아래 예제를 참고하자.)</span>


```cpp
map<vector<bool>,int> cache;
```
<br/>

아래는 같은 역할을 하는 코드이다.

```cpp
int cache[];
```

## 비트 연산

> 기본

```cpp
a & b    // AND    000101 & 000011 = 000001
a | b    // OR     000101 |  000011 = 000111
a ^ b    // XOR    000101 ^ 000011 = 000110
~a       // NOT   ~000101 = 111010
a << b   // SHIFT  000101 << 000011 = 101000
a >> b   // SHIFT  000101 >> 000011 = 000000
```
<br/>

> 자주 하는 실수

<span class="paragraph_bolster">연산자 우선순위</span> 망각하기
```cpp
bool a = (6 & 3 == 2);     // a = 1이라고 착각. 비교연산자 "==" 이 먼저 수행되기 때문에 a = 0.
```
<br/>

<span class="paragraph_bolster">정수 오버플로우</span> 간과하기
```cpp
bool isFlagUp(unsigned long long a, int idx) {
  return (a & (1 << idx)) > 0;    // 1은 int형. idx가 32보다 크면 오버플로우가 발생
}
```
<br/>
올바른 구현:
```cpp
bool isFlagUp(unsigned long long a, int idx) {
  return (a & (1ull << idx)) > 0;  // 여전한 제약조건: idx < 64
}
```
